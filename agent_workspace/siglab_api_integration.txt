SoDEX API - Signed Writes (siglab/live/sodex_signing.py + sodex_client.py)
-------------------------------------------------------------------------
Auth: EIP-712 typed data over Keccak-256 of canonical JSON.
  Headers attached after signing: X-API-Key, X-API-Sign, X-API-Nonce
  (per module-live-boundary.md:218-224)

Envs checked by sodex-preflight (siglab/cli/helpers.py:157-220):
  SODEX_API_KEY_NAME        (required)
  SODEX_ACCOUNT_ID          (required, must be unsigned int)
  SODEX_NONCE_STORE_PATH    (required, JSON file, parent writable, parseable)
  SODEX_PRIVATE_KEY         (EVM private key)
  SODEX_ENVIRONMENT         (default = "testnet", choices: testnet | mainnet)

Mainnet double-confirmation (docs/module-live-boundary.md:380-401):
  BOTH must be set to allow mainnet live write:
    SODEX_TESTNET_PREFLIGHT_PASSED=true
    SODEX_MAINNET_LIVE_WRITE_CONFIRMATION=I_UNDERSTAND_MAINNET_RISK

Signed-write kinds (siglab/cli/sodex.py sodex-preview, dry-run only):
  new-order          --symbol-id, --cl-ord-id, --modifier (NORMAL/STOP/BRACKET/ATTACHED_STOP),
                      --side (BUY/SELL), --order-type (LIMIT/MARKET),
                      --time-in-force (GTC/FOK/IOC/GTX), --price, --quantity, --funds
  cancel-order       --order-id, --orig-cl-ord-id
  schedule-cancel    --scheduled-timestamp
  update-leverage    --amount
  update-margin      --amount

Signing path: siglab/live/sodex_signing.py
  1. Serialize request body as canonical JSON
  2. Keccak-256 hash the canonical JSON -> payloadHash
  3. Build EIP-712 typed data (ExchangeAction struct with payloadHash + nonce)
  4. Sign with EIP-712 (encode_typed_data -> Account.sign_message)
  5. Prefix signature with 0x01 (EIP-712 indicator)
  6. Attach headers X-API-Key, X-API-Sign, X-API-Nonce

Nonce store: persists monotonically increasing nonces per API key to JSON file.
  - 2-day past / 1-day future window
  - Rejects duplicate nonces
  - High-water mark tracking (max 64 nonces per key)

Preflight gate (siglab/cli/sodex.py):
  siglab sodex-preflight [--json] [--exit-on-first-frame]
    Reads 4 envs + checks documented endpoint weights.
    Returns: {environment, public_read_ready, signed_path{signer_ready, accountID_present,
              api_key_name_present, nonce_store_ready, ...}, live_write_allowed,
              live_write_refusal_reason, access_plan, next_actions,
              request_weight_budget_per_minute, documented_endpoint_weights,
              rate_limit_scope, supported_signed_actions, unsupported_signed_actions}

  --exit-on-first-frame (added Wave K1): short-circuit after env reads, no network.

  siglab sodex-preview --kind <new-order|cancel-order|schedule-cancel|update-leverage|update-margin>
    Dry-run signing payload builder. Does NOT submit. Does NOT call build_signature.
    Output: {signature: None, submitted: False} (siglab/cli/sodex.py:289-290)

  REFUSED unless preflight reports live_write_allowed=true.


SoDEX API - Public Reads (siglab/data/sodex_feeds.py + sodex_ws.py)
------------------------------------------------------------------
HTTP base URL: https://mainnet-gw.sodex.dev/api/v1/perps  (default; mainnet-first)
WSS URLs (4 combinations in siglab/live/sodex_ws.py:59-64):
  wss://mainnet-gw.sodex.dev/ws/{spot,perps}
  wss://testnet-gw.sodex.dev/ws/{spot,perps}

SoDEXPublicPerpsClient (siglab/data/sodex_client.py) - read-only, no auth.
All endpoints under /markets/ (mainnet default base):
  GET /markets/symbols                          -> symbols()
  GET /markets/coins                            -> coins()
  GET /markets/tickers                          -> tickers()
  GET /markets/miniTickers                      -> mini_tickers()
  GET /markets/mark-prices                      -> mark_prices()
  GET /markets/bookTickers                      -> book_tickers()
  GET /markets/{symbol}/orderbook               -> orderbook(symbol, limit)
  GET /markets/{symbol}/klines                  -> klines(symbol, interval)
  GET /markets/{symbol}/trades                  -> trades(symbol, limit)
  GET /markets/{symbol}/fundingRate             -> funding_history(symbol, ...)
  GET /accounts/{user}/balances                 -> account_balances(user_address, account_id)
  GET /accounts/{user}/orders                   -> account_orders(user_address, ...)
  GET /accounts/{user}/positions                -> account_positions(user_address, account_id)
  GET /accounts/{user}/state                    -> account_state(user_address, account_id)

SoDEXFeeds (siglab/data/sodex_feeds.py) - high-level wrapper with ParquetLake caching:
  fetch_klines, fetch_symbols, fetch_tickers, fetch_mark_prices,
  fetch_book_tickers, fetch_orderbook, fetch_trades (all REST, no WSS)

SoDEXWebSocketClient (siglab/live/sodex_ws.py):
  Default environment = "mainnet", market = "perps" (line 112-113).
  WSS public channels: allBookTicker, bookTicker, markPrice, kline, depth, trade, allMids
  WSS ACCOUNT channels (require user=EVM address + accountID):
    orderUpdate, accountOrders, accountTrades, accountPositions
  Validation: SODEX_WS_CHANNELS whitelist (sodex_ws.py:66-)
  Auth: NONE on public channels. Account channels: preflight-validated params only;
        live signed validation still UNPROVEN (per AGENTS.md).

Probe: siglab/cli/sodex.py sodex-ws-probe
  --environment [mainnet|testnet]  default=mainnet
  --market [spot|perps]            default=perps
  --channel                        default=allBookTicker
  --symbol, --user-address, --account-id, --timeout-seconds (default=8.0)
  --evidence-output, --json
  --exit-on-first-frame  (added Wave 1): exits on first valid frame, sub-second probe
  Hardcodes: live_write=False, signed=False (read-only, never signs)

Testnet-First Discipline (docs/access-and-testnet-plan.md)
==========================================================
  SoDEX signed writes -> testnet FIRST, mainnet after explicit confirmation.
  SoDEX public reads  -> mainnet by DEFAULT, testnet opt-in.
  SoDEX WSS           -> mainnet by DEFAULT (sodex_ws.py:112).
  Public market data  -> no auth on public channels.
  Private/account WS  -> requires user (EVM addr) + accountID, but still
                         unvalidated per AGENTS.md:62.

Live boundary rules (per AGENTS.md and module-live-boundary.md):
  - SoSoValue calls use x-soso-api-key (no other auth).
  - SoDEX signed writes REFUSE without account_id + api_key_name +
    nonce store + private key.
  - Mainnet signed writes REQUIRE SODEX_TESTNET_PREFLIGHT_PASSED=true
    AND SODEX_MAINNET_LIVE_WRITE_CONFIRMATION=I_UNDERSTAND_MAINNET_RISK.
  - SoDEX WSS public is NOT private/account stream ready.
  - ValueChain chain-id preflight is read-only readiness, not execution.
  - B.AI Credits are not USD.
