## CRITICAL

### C-01. `get_sodex_feeds()` method does not exist — 4 market routes return HTTP 500

**File:** `siglab/dashboard/routes.py`
**Lines:** 1990, 2003, 2021, 2050
**Routes affected:** `/market/symbols`, `/market/tickers`, `/market/klines/{symbol}`, `/market/orderbook/{symbol}`

All four market routes call `request.app.state.dashboard.get_sodex_feeds()` which is NOT defined on `DashboardState`. The try/except blocks only catch `(httpx.HTTPError, OSError, ValueError)`, not `AttributeError`. Docs mention this method as lazy-initialized but it was never implemented. All market data features are completely broken.

### C-02. ETF evidence pipeline emits all-null values (camelCase/snake_case mismatch)

**Files:** `siglab/data/feeds.py:1333-1349`, `siglab/data/feeds.py:1851-1865`, `siglab/data/evidence.py:176-191`

The ETF API returns camelCase keys (`totalNetInflow`, `totalValueTraded`, etc.) but `etf_historical_inflow()` specifies `required_fields` in snake_case (`total_net_inflow`, etc.). `_fill_optional_fields` silently sets all mismatched fields to `None`. Every ETF evidence record has `value: null`.

### C-03. `evidence-build` CLI command is dead code — never registered

**File:** `siglab/cli/__init__.py:24-36`

`siglab/cli/evidence.py` has `add_subparser()` and `run_evidence_build()` but is never imported in the CLI entry point. The command is completely unreachable.

### C-04. `/api/search` uses wrong field names — run search never returns results

**File:** `siglab/dashboard/routes.py`
**Lines:** 1498-1501

- `run.get("run_id", "")` — actual field is `run_session_id`
- `run.get("label", "")` — actual field is `run_label`
- `run.get("family", "")` — actual field is `families` (plural list)
- `exp.get("hypothesis", "")` — experiments have no top-level `hypothesis` field

Searching for run IDs, labels, or families never returns results.

### C-05. `sparklineSvg()` in chart-engine.js references undefined `svg` variable

**File:** `siglab/dashboard/static/chart-engine.js:100`

`sparklineSvg()` does not define `svg` anywhere in scope. At line 100, `svg.parentElement` throws `ReferenceError`. This crashes dashboard run card rendering when series data has finite values.

## HIGH

### H-01. No separate ETF/news base URL config fields

**Files:** `siglab/config.py:25-51`, `siglab/dashboard/routes.py:109-110,1411-1412`, `siglab/data/feeds.py:220-226`, `siglab/cli/evidence.py:60-64`

`SiglabConfig` has only `sosovalue_base_url`. Documented `SOSOVALUE_ETF_BASE_URL` and `SOSOVALUE_NEWS_BASE_URL` env vars have zero effect — no fields consume them.

### H-02. `config.json` API key never consumed

**Files:** `siglab/config.py:90-112`, `siglab/cli/helpers.py:372-384`

`config.json` has an `api_key` but `load_settings()` only reads `SOSOVALUE_API_KEY` env var. `require_sosovalue_config()` only checks file existence, never reads the key.

### H-03. Series endpoint (`/api/experiments/{hash}/series`) lacks error handling

**File:** `siglab/dashboard/routes.py:1458-1463`

No try/except wrapping. If `experiment_series_payload()` raises (NaN serialization, artifact loading failure, state corruption), it becomes an unhandled 500.

### H-04. WebSocket endpoint (`/ws`) lacks connection-level error handling

**File:** `siglab/dashboard/routes.py:2661-2721`

If anything fails before/during `websocket.accept()`, the error propagates as HTTP 500 WebSocket rejection.

### H-05. Search empty query missing `count` field in response

**File:** `siglab/dashboard/routes.py:1489-1490`

When `q` is empty, the early return omits the `count` field that the non-empty path includes. API consumers expecting `count` get inconsistent responses.

**Lines:** 165, 886, 889, 898, 1010, 1118, 1131, 1157, 1189, 1221, 1226, 1267, 1280, 1284, 1286, 1309

**File:** `siglab/dashboard/routes.py:2812-2813`

Module-level `_WS_HANDLERS` dict cleared on every `_handle_message()` call. Two concurrent WebSocket connections interfere with each other's handlers.

8 experiments in DB but all have `artifact_path = NULL`. Series endpoint returns `series_available: false`. Canonical runs, equity curves, metrics charts never shown.

**File:** `siglab/data/feeds.py:1241`, `docs/module-orchestration.md:267`
